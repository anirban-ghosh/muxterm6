# SFTP Browser: Symlinks, Context Menu, Column Sorting

## Context

Three issues with the SFTP browser need fixing:
1. **Remote symlinks not followed** — `list()` checks `item.type === 'd'` but symlinks have type `'l'`, so symlinked directories appear as files and can't be navigated
2. **No right-click context menu** — clipboard/delete/transfer operations only work via keyboard shortcuts
3. **No column sorting** — files are always sorted dirs-first then alphabetically by name, with no way to sort by size, date, or permissions

---

## Changes

### 1. Follow symlinks in remote listing

**File:** `src/main/sftp/sftp-connection-manager.ts` — `list()` method (line 135)

The ssh2-sftp-client `list()` returns `type: 'l'` for symlinks. For each symlink entry, call `client.stat(fullPath)` which follows symlinks and returns resolved attributes. Use the resolved stat to determine `isDirectory`.

```
For each item in listing:
  if item.type === 'l':
    try { stat = await client.stat(fullPath); isDirectory = stat.isDirectory }
    catch { isDirectory = false }  // broken symlink → treat as file
  else:
    isDirectory = item.type === 'd'
```

**File:** `tests/unit/sftp-connection-manager.spec.ts` — add test for symlink handling in `list()`.

### 2. Right-click context menu in FileBrowser

Close menu on: click anywhere, scroll, Escape key.

### 3. Column sorting

**File:** `src/renderer/components/SftpBrowser/FileBrowser.tsx`

Add local state: `sortColumn: 'name' | 'size' | 'date' | 'perms'` (default `'name'`), `sortDirection: 'asc' | 'desc'` (default `'asc'`).

Make column headers clickable:
- Click same column → toggle direction
- Click different column → set column, reset to asc

Sort files in `useMemo`:
- Directories always grouped first (regardless of sort)
- Within each group, sort by selected column and direction
- Name: `localeCompare`; Size: numeric; Date: numeric; Perms: string compare

Show sort indicator (▲/▼) next to active column header text.

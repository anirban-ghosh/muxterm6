# SFTP Browser — Test Suite Plan

## Context

The SFTP Browser feature was implemented without tests. This plan covers a comprehensive unit test suite for all SFTP-related code. Tests follow existing project conventions: Vitest in `tests/unit/`, globals mode, `@shared` alias, `vi.mock()` for dependencies.

---

## Test Files to Create (6 files)

### 1. `tests/unit/sftp-ssh-config-parser.spec.ts`
**Tests:** `src/main/sftp/ssh-config-parser.ts`

| Test | What it verifies |
|------|-----------------|
| Parses basic host entry | hostname, port, user, identityFile all extracted |
| Parses multiple hosts | Returns array with all entries |
| Skips wildcard hosts | `Host *` and `Host *.example.com` excluded |
| Handles missing fields | Defaults: port=22, user='', no identityFile |
| Expands tilde in IdentityFile | `~/` replaced with homedir |
| Skips comments and blank lines | `#` lines and empty lines ignored |
| Returns empty array when config missing | Graceful ENOENT handling |
| Multi-word Host line | Takes first token only |
| Case-insensitive keywords | `HostName` and `hostname` both work |

**Mocking:** `vi.mock('fs/promises')` — mock `readFile`.
